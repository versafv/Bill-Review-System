# ZSC 管理系统 - 运行环境搭建指南

## 一、项目简介

ZSC 是一套基于 SpringBoot3 + Vue3 的前后端分离的快速开发框架，本书库管理系统是其典型应用。

### 技术栈
- **后端**: Spring Boot 3.5.8 + MyBatis-Plus + Redis + JWT
- **前端**: Vue 3 + Vite + Element Plus + Pinia
- **数据库**: MySQL 8.0+
- **JDK 版本**: JDK 17
- **构建工具**: Maven + Node.js

---

## 二、环境准备

### 2.1 必需软件安装

#### 1. JDK 17
- **下载地址**: https://www.oracle.com/java/technologies/javase/jdk17-archive-downloads.html
- **安装步骤**:
  1. 下载 JDK 17 Windows x64 安装包
  2. 双击安装，建议安装路径：`C:\Program Files\Java\jdk-17`
  3. 配置环境变量:
     - 新建 `JAVA_HOME` = `C:\Program Files\Java\jdk-17`
     - 编辑 `Path`，添加 `%JAVA_HOME%\bin`
  4. 验证安装:
     ```bash
     java -version
     ```
     应显示 `java version "17.x.x"`

#### 2. Maven 3.6+
- **下载地址**: https://maven.apache.org/download.cgi
- **安装步骤**:
  1. 下载 `apache-maven-3.8.x-bin.zip`
  2. 解压到无中文路径，如：`D:\dev\apache-maven-3.8.6`
  3. 配置环境变量:
     - 新建 `MAVEN_HOME` = `D:\dev\apache-maven-3.8.6`
     - 编辑 `Path`，添加 `%MAVEN_HOME%\bin`
  4. 验证安装:
     ```bash
     mvn -version
     ```

#### 3. MySQL 8.0+
- **下载地址**: https://dev.mysql.com/downloads/mysql/
- **安装步骤**:
  1. 下载 MySQL Installer for Windows
  2. 选择 Server only 或 Developer Default 安装
  3. 设置 root 用户密码（请牢记）
  4. 默认端口：3306
  5. 验证安装:
     ```bash
     mysql -u root -p
     ```

#### 4. Redis (Windows 版本)
- **GitHub 地址**: https://github.com/microsoftarchive/redis/releases
- **下载**: `Redis-x64-3.2.100.msi`
- **安装步骤**:
  1. 双击安装，默认端口 6379
  2. 启动 Redis 服务:
     ```bash
     redis-server.exe redis.windows.conf
     ```
  3. 验证安装:
     ```bash
     redis-cli ping
     ```
     返回 `PONG` 表示成功

#### 5. Node.js 22+
- **下载地址**: https://nodejs.org/
- **推荐版本**: v22.22.2 LTS 或更高
- **安装步骤**:
  1. 下载并双击安装
  2. 验证安装:
     ```bash
     node -version
     npm -version
     ```

#### 6. Git
- **下载地址**: https://git-scm.com/
- 用于代码版本管理

## 三、数据库初始化

### 3.1 创建数据库

1. 登录 MySQL:
   ```bash
   mysql -u root -p
   ```

2. 执行以下 SQL 创建数据库:
   ```sql
   -- 创建系统数据库
   CREATE DATABASE IF NOT EXISTS `zsc` DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_0900_ai_ci;
   
   USE `zsc`;
   ```

### 3.2 导入数据表

#### 方法一：使用命令行
导入系统表:  
   ```bash
   mysql -u root -p zsc < D:\Git\zsc_backend\sql\zsc.sql
   ```

#### 方法二：使用 Navicat

1. 打开 Navicat
2. 连接到 MySQL 服务器
3. 右键点击 `zsc` 数据库 → 运行 SQL 文件
4. 选择 `sql/zsc.sql`

## 四、后端配置与启动

### 4.1 修改数据库配置

编辑文件：`zsc-admin/src/main/resources/application-druid.yml`

```yaml
spring:
    datasource:
        druid:
            master:
                url: jdbc:mysql://localhost:3306/zsc?useUnicode=true&characterEncoding=utf8&zeroDateTimeBehavior=convertToNull&useSSL=true&serverTimezone=GMT%2B8
                username: root          # 修改为你的 MySQL 用户名
                password: 123123        # 修改为你的 MySQL 密码
```

**重要**: 
- 确保数据库名、用户名、密码与你的 MySQL 配置一致
- 如果 Redis 有密码，需修改 `application.yml` 中的 Redis 配置

### 4.2 Maven 编译打包

1. 打开命令行（PowerShell 或 CMD）
2. 进入项目根目录:
   ```bash
   cd D:\Git\zsc_backend
   ```

3. 清理编译（可选）:
   ```bash
   mvn clean
   ```

4. 编译打包:
   ```bash
   mvn package -DskipTests
   ```
   
   或使用阿里云镜像加速:
   ```bash
   mvn package -DskipTests -Dmaven.repo.local=D:\dev\.m2\repository
   ```

5. 编译成功后，会在各模块的 `target` 目录下生成 jar 包

### 4.3 启动后端服务

#### 方式一：使用 IDEA 启动

1. 用 IDEA 打开项目
2. 找到 `zsc-admin/src/main/java/com/zsc/ZscApplication.java`
3. 右键 → Run 'ZscApplication'

#### 方式二：使用命令行启动

```bash
cd zsc-admin/target
java -jar zsc-admin.jar
```

观察控制台输出，出现以下日志表示启动成功:

## 五、前端配置与启动

### 5.1 安装依赖

1. 进入前端目录:
   ```bash
   cd D:\Git\zsc_backend\zsc-frontend
   ```

2. 安装 Node.js 依赖（首次运行需要）:
   ```bash
   npm install
   ```
   
   或使用淘宝镜像加速:
   ```bash
   npm config set registry https://registry.npmmirror.com
   npm install
   ```

   **常见问题**:
   - 如果安装失败，检查 Node.js 版本是否 >= 22
   - 删除 `node_modules` 和 `package-lock.json` 重新安装

### 5.2 配置开发环境

编辑文件：`zsc-frontend/.env.development`

```bash
# 页面标题
VITE_APP_TITLE = ZSC 管理系统

# 开发环境配置
VITE_APP_ENV = 'development'

# 管理系统/开发环境
VITE_APP_BASE_API = '/dev-api'
```

通常无需修改，保持默认即可。

### 5.3 启动前端开发服务器

#### 方式一：使用命令行

```bash
cd zsc-frontend
npm run dev
```

#### 方式二：使用 IDEA 启动

1. 用 IDEA 打开项目
2. 在IDEA右上角当前文件 → 编辑配置 → 添加配置 → 选择npm → 脚本选dev
3. 点击绿色三角运行

### 5.4 验证前端启动成功

观察控制台输出，出现以下信息表示成功:
```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:80/
  ➜  Network: use --host to expose
```

访问前端:
- 开发环境地址：http://localhost:80
- 默认账号：`admin` / `admin123`

## 六、前后端联调

### 6.1 代理配置说明

前端开发时，通过 Vite 代理将请求转发到后端:

配置文件：`zsc-frontend/vite.config.js`

```javascript
proxy: {
  '/dev-api': {
    target: 'http://localhost:8080',
    changeOrigin: true,
    rewrite: (path) => path.replace(/^\/dev-api/, '')
  }
}
```

### 6.2 跨域问题解决

如果出现跨域错误，检查:
1. 后端是否启动成功
2. 前端代理配置是否正确
3. 浏览器控制台是否有 CORS 相关错误

## 八、常见问题排查

### 8.1 后端启动失败

**问题 1**: 数据库连接失败
```
com.mysql.cj.jdbc.exceptions.CommunicationsException: Communications link failure
```
**解决**:
- 检查 MySQL 服务是否启动
- 确认数据库名、用户名、密码正确
- 检查端口 3306 是否被占用

**问题 2**: Redis 连接失败
```
io.lettuce.core.RedisConnectionException: Unable to connect to localhost:6379
```
**解决**:
- 启动 Redis 服务
- 检查 Redis 是否正常运行：`redis-cli ping`

**问题 3**: 端口被占用
```
Port 8080 was already in use
```
**解决**:
- 修改 `application.yml` 中的端口号
- 或关闭占用 8080 端口的程序

### 8.2 前端启动失败

**问题 1**: npm install 失败
```
npm ERR! network timeout
```
**解决**:
```bash
npm config set registry https://registry.npmmirror.com
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```

**问题 2**: 运行时错误
```
Error: Cannot find module 'xxx'
```
**解决**:
- 删除 `node_modules` 和 `package-lock.json`
- 重新执行 `npm install`

**问题 3**: 热更新不生效
**解决**:
- 重启开发服务器
- 清除浏览器缓存

### 8.3 登录失败

**问题**: 登录后提示"用户名或密码错误"
**解决**:
1. 检查数据库是否正确导入
2. 查看 `sys_user` 表中是否有 admin 用户
3. 确认密码加密方式是否正确（默认密码：admin123）

### 8.4 接口请求失败

**问题**: 前端请求后端接口 404 或 500
**解决**:
1. 检查后端是否正常运行
2. 查看后端日志是否有异常
3. 确认请求路径是否正确
4. 检查跨域配置
