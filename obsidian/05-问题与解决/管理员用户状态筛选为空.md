# 管理员用户状态筛选为空

## 问题描述

管理员用户管理页面，选择状态（正常/停用）后点击查询，返回全部用户而非筛选结果。后端日志显示 `status` 参数为空字符串或 null。

## 复现步骤

1. 管理员登录，进入「用户管理」页面
2. 状态下拉选择「停用」
3. 点击搜索按钮
4. 返回全部 7 条用户数据，未按停用状态筛选

## 原因分析

问题出在 **`request.js` 请求拦截器** 的 GET 参数序列化逻辑（第 35-40 行）：

```js
if (config.method === 'get' && config.params) {
    let url = config.url + '?' + tansParams(config.params)
    url = url.slice(0, -1)   // ← BUG!
    config.params = {}
    config.url = url
}
```

`tansParams()` 的返回值格式为 `key1=value1&key2=value2`，**没有尾部字符**（内部已通过 `.substring(1)` 去掉了前导 `&`）。

但 `.slice(0, -1)` 始终砍掉拼接后 URL 的最后一位字符：

| `tansParams` 输出 | 拼接后 | `.slice(0,-1)` 后 | 结果 |
|---|---|---|---|
| `status=1` | `/users?status=1` | `/users?status=` | **值被截断** |
| `userName=test` | `/users?userName=test` | `/users?userName=tes` | **值被截断** |
| `""` (无参数) | `/users?` | `/users` | 正常 |

该行代码本意是处理空参数时的多余 `?`，但副作用是把所有带参数请求的最后一个字符切掉了。

## 解决方案

判断 `tansParams` 返回值，有值才拼 `?`，去掉 `.slice(0, -1)`：

```js
if (config.method === 'get' && config.params) {
    const qs = tansParams(config.params)
    config.url = qs ? config.url + '?' + qs : config.url
    config.params = {}
}
```
