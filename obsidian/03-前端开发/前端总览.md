# 前端总览

基于 **RuoYi-Vue** 前端模板的 Vue 3 SPA 应用。

## 技术栈

| 技术 | 版本 | 说明 |
|------|------|------|
| Vue | 3.5.26 | Composition API + `<script setup>` |
| Vite | 5 | 构建工具，HMR 热更新 |
| Element Plus | 2.13.1 | UI 组件库，全局注册 |
| Vue Router | 4.6.4 | history 模式路由 |
| Pinia | 3.0.4 | 状态管理 |
| Axios | 1.13.2 | HTTP 请求 |
| Sass | sass-embedded 1.97.2 | CSS 预处理 |
| ECharts | 5.6.0 | 图表 |

## 三层权限模型

| 层级 | 机制 | 示例 |
|------|------|------|
| 路由 | 路由 meta 中的 `roles` / `permissions` | `permissions: ['system:user:list']` |
| 按钮/元素 | `v-hasPermi` / `v-hasRole` 指令 | `v-hasPermi="['biz:category:add']"` |
| API | 后端 `@PreAuthorize` | Java Controller 方法注解 |

## 登录到动态路由流程

```
/login → POST /login (返回 token)
  → Cookie 存储 Admin-Token
  → 路由守卫检测 token → userStore.getInfo() → GET /getInfo (获取 roles, permissions)
  → permissionStore.generateRoutes()
    → GET /getRouters (后端菜单 JSON 树)
    → import.meta.glob('@/views/**/*.vue') 动态匹配组件
    → filterDynamicRoutes() 检查权限
    → router.addRoute() 注册动态路由
    → 可访问页面渲染
```

## Axios 请求链路

```
页面 → api/xxx.js → Axios 实例
  → Request 拦截器:
    - 注入 Authorization: Bearer <token>
    - GET 参数序列化
    - POST/PUT 防重复提交 (5MB 限制, 1s 间隔, sessionStorage)
  → Vite Proxy: 剥离 /dev-api 前缀 → http://localhost:8080
  → Response 拦截器:
    - 200: resolve
    - 401: 弹出重新登录对话框
    - 500: 错误提示
    - 601: 警告提示
```

## 三种导航模式

| 模式 | navType | 说明 |
|------|---------|------|
| 侧边栏导航 | 1 (默认) | 左侧树形菜单，顶部 面包屑 |
| 混合导航 | 2 | 侧边栏 + 顶部一级菜单 |
| 顶部导航 | 3 | 纯顶部水平菜单 |

## 布局自适应

- ≥ 992px: 桌面模式（侧边栏展开）
- < 992px: 移动模式（侧边栏隐藏，hamburger 切换）
- 暗黑模式切换：`startViewTransition` API 实现平滑过渡
