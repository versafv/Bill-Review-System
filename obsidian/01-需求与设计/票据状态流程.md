# 票据状态流程

## 状态说明

| 状态 | 标识 | 说明 | 可执行操作 |
|------|------|------|------------|
| 草稿 | 0 | 临时保存，未提交 | 编辑、删除、提交 |
| 已提交 | 1 | 已提交，待审核 | 查看（不可编辑） |
| 已通过 | 2 | 管理员审核通过 | 查看（终态） |
| 已退回 | 3 | 管理员审核退回 | 编辑、重新提交 |

## 状态流转

```mermaid
stateDiagram-v2
    [*] --> 草稿 : 保存草稿
    草稿 --> 草稿 : 编辑
    草稿 --> [*] : 删除
    草稿 --> 已提交 : 提交审核
    已提交 --> 已通过 : 管理员通过
    已提交 --> 已退回 : 管理员退回
    已退回 --> 已退回 : 编辑
    已退回 --> 已提交 : 重新提交
    已通过 --> [*]
```

## 详细状态流

```mermaid
flowchart TD
    A[用户进入票据表单] --> B{操作选择}
    B -->|保存草稿| C[status=0 草稿]
    B -->|直接提交| D[status=1 已提交]

    C --> E{用户操作}
    E -->|编辑| C
    E -->|删除| F[票据删除]
    E -->|提交审核| D

    D --> G[管理员查看票据]
    G --> H{审核决定}
    H -->|通过| I[status=2 已通过<br/>记录 audit_by, audit_time, audit_comment]
    H -->|退回| J[status=3 已退回<br/>记录 audit_by, audit_time, audit_comment]

    J --> K[用户查看退回意见]
    K --> L{用户选择}
    L -->|编辑票据| C
    L -->|重新提交| D
    L -->|放弃| F

    I --> M[结束]
    F --> M
```

## 业务规则

```mermaid
flowchart LR
    subgraph 编辑规则
        direction TB
        R1[草稿状态: 可编辑所有字段]
        R2[已提交状态: 不可编辑]
        R3[已通过状态: 不可编辑]
        R4[已退回状态: 可编辑<br/>编辑后可重新提交]
    end

    subgraph 删除规则
        direction TB
        D1[草稿状态: 可删除]
        D2[已提交/已通过/已退回: 不可删除]
    end

    subgraph 查看规则
        direction TB
        V1[本人票据: 可查看详情]
        V2[管理员: 可查看所有票据]
    end
```

| 编辑 | ✓ | ✗ | ✗ | ✓ |

| 提交审核 | ✓ | ✗ | ✗ | — |
| 重新提交 | ✗ | ✗ | ✗ | ✓ |
