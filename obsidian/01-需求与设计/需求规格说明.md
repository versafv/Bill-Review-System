# 需求规格说明

> 完整文档: [需求规格说明书.md](../../docs/需求规格说明书.md)

## 1. 项目背景

基于 RuoYi-Vue（Spring Boot + Vue 3）框架进行二次开发，实现一个支持多角色的在线票据提交与审核系统。目标用户为组织内部员工（提交票据）与管理员（审核票据、管理用户）。

## 2. 术语定义

| 术语 | 说明 |
|------|------|
| 票据 | 用户提交的报销/差旅/办公等凭证，包含文字信息和附件 |
| 草稿 | 票据的临时保存状态，可反复修改，可删除 |
| 提交 | 将草稿票据正式提交进入审核流程，提交后不可再修改 |
| 审核 | 管理员对已提交票据做出通过或退回的决定 |
| 普通用户 | 只能查看和操作自己票据的用户 |
| 管理员 | 拥有用户管理和票据审核权限的用户 |

## 3. 系统角色

| 角色 | 权限概述 | 获得方式 |
|------|----------|----------|
| 普通用户 | 管理个人票据（增/删/改/查），查看个人票据状态 | 自行注册 或 管理员添加 |
| 管理员 | 用户管理（CRUD/密码重置/冻结），票据审核（通过/退回） | 系统预置 |

## 4. 功能需求

### 4.1 普通用户功能

#### FR-01 用户注册

- 用户可通过注册页面自行注册账号
- 输入: 用户名、密码、确认密码、验证码
- 注册成功自动分配普通用户角色
- 复用框架已有注册功能

#### FR-02 添加票据

- 用户填写票据信息并选择"保存草稿"或"提交"
- 必填: 票据标题、金额
- 选填: 票据类别、描述、附件（多文件）
- 保存草稿: status=0，billNo 为临时编号（DRAFT- + 时间戳），可后续编辑
- 直接提交: status=1，生成正式编号 BILL-YYYYMMDD-NNNN

#### FR-03 修改草稿票据

- 前置条件: 票据 status=0（草稿）或 status=3（已退回），且为本人票据
- 约束: 已提交/已通过的票据不可修改

#### FR-04 删除草稿票据

- 前置条件: 票据 status=0（草稿），且为本人票据
- 票据及关联附件一并删除
- 约束: 已提交的票据不可删除

#### FR-05 提交草稿票据

- 将草稿或已退回票据正式提交进入审核流程
- status: 0 或 3 → 1，生成正式票据编号，临时编号 DRAFT- 前缀的替换掉
- 提交后不可撤回、不可修改

#### FR-06 查看已通过票据

- 用户查看自己已通过审核的票据列表及详情（status=2）

#### FR-07 查看未审核票据

- 用户查看自己已提交但尚未审核的票据列表及详情（status=1）

#### FR-08 查看个人所有票据

- 按状态分类展示: 全部 / 草稿 / 待审核 / 已通过 / 已退回
- 分页加载，默认每页 10 条

#### FR-09 票据查询

- 查询条件: 关键词（标题）、票据类别、日期范围
- 输出分页列表: 票据编号、标题、类别名称、金额、状态、创建时间
- 约束: 只能查看本人提交的票据

#### FR-10 查看票据详情

- 查看票据完整信息，含附件列表、审核历史（多次审核记录）

### 4.2 管理员功能

#### FR-11 用户管理

- 对系统用户进行增删改查
- 功能: 新增用户、编辑用户、删除用户、重置密码、冻结/解冻账号
- 复用框架已有功能

#### FR-12 票据审核

- 对已提交的票据做出审核决定
- 输入: 审核结果（通过/退回）、审核意见
- 通过: status → 2，记录 auditBy 和 auditTime
- 退回: status → 3，记录 auditBy 和 auditTime
- 前置条件: 票据 status=1（已提交）

#### FR-13 查看所有待审核票据

- 管理员查看系统中所有 status=1 的票据

#### FR-14 查看票据详情

- 管理员可查看任一票据的完整信息，含附件和提交人信息
- 管理员约束: 不能删除票据

## 5. 票据状态与操作矩阵

| 状态 | 标识 | 可编辑 | 可删除 | 可审核 | 可查看 |
|------|------|:------:|:------:|:------:|:------:|
| 草稿 | 0 | 是 | 是 | 否 | 是 |
| 已提交 | 1 | 否 | 否 | 是 | 是 |
| 已通过 | 2 | 否 | 否 | 否 | 是 |
| 已退回 | 3 | 是 | 否 | 否 | 是 |

## 6. 非功能需求

### 安全性
- 所有接口需 JWT Token 认证，未登录不可访问
- 权限控制到按钮级: 普通用户不能看到管理员专属按钮
- 后端校验数据归属: 用户只能操作自己的票据
- 密码 BCrypt 加密存储

### 可用性
- 状态以不同颜色标签区分: 草稿(灰) / 已提交(蓝) / 已通过(绿) / 已退回(红)
- 操作结果有明确提示（成功/失败消息）

### 性能
- 票据列表分页加载，默认每页 10 条
- 附件上传单文件限制 10MB

### 可扩展性
- 票据类别通过 biz_category 表管理，可后续扩展
- 审核流程通过 biz_audit_log 表支持多级审核扩展

## 7. 数据需求

### 票据数据

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| 票据编号 | 字符串 | 是 | 系统自动生成: BILL-YYYYMMDD-NNNN |
| 标题 | 字符串 | 是 | 最长 200 字符 |
| 票据类别 | 外键 | 否 | 关联 biz_category 表 |
| 金额 | 数字 | 是 | 精确到小数点后两位 |
| 描述 | 文本 | 否 | 长文本 |
| 状态 | 枚举 | 是 | 0-草稿 1-已提交 2-已通过 3-已退回 |
| 附件 | 文件列表 | 否 | 支持多个文件 |

### 附件数据

| 字段 | 类型 | 说明 |
|------|------|------|
| 文件名 | 字符串 | 原始文件名 |
| 文件路径 | 字符串 | 服务器存储路径 |
| 文件大小 | 数字 | 字节数 |
| 上传时间 | 日期时间 | 上传时刻 |

## 8. 约束与假设

1. 基于已有 RuoYi-Vue 框架，用户管理、登录认证、权限控制等复用框架
2. 服务器环境: JDK 17 + MySQL 8.0 + Redis
3. 浏览器环境: Chrome / Firefox / Edge 现代版本
4. 单次附件上传不超过 10MB
5. 票据审核为单级审核，不涉及多级审批流
6. 用户标识字段 `create_by` 使用 RuoYi 框架约定的用户名（varchar），不使用 userId

## 9. 验收标准

| 编号 | 测试场景 | 预期结果 |
|------|----------|----------|
| AC-01 | 用户注册新账号 | 注册成功，登录后可访问"我的票据" |
| AC-02 | 用户新建票据，保存草稿 | status=0，可再次编辑 |
| AC-03 | 用户提交草稿票据 | status→1，生成编号，编辑按钮消失 |
| AC-04 | 用户尝试修改已提交票据 | 操作被拒绝 |
| AC-05 | 用户删除草稿票据 | 票据及附件被删除 |
| AC-06 | 用户查看票据列表 | 仅显示本人的票据 |
| AC-07 | 用户按关键词搜索票据 | 返回匹配的票据 |
| AC-08 | 管理员查看待审核票据 | 显示所有已提交的票据 |
| AC-09 | 管理员审核通过票据 | status→2 |
| AC-10 | 管理员审核退回票据 | status→3，含退回意见 |
| AC-11 | 退回后编辑并重新提交 | 可编辑，可重新提交 |
| AC-12 | 普通用户访问审核页面 | 无权限，页面不显示 |
| AC-13 | 未登录访问任意接口 | 返回 401 未授权 |
