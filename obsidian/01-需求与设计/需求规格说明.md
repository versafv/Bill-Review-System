# 需求规格说明

> 完整文档: [[../docs/需求规格说明书|需求规格说明书.md]]

## 系统角色

| 角色 | 权限概述 |
|------|----------|
| 普通用户 | 管理个人票据（增/删/改/查），查看个人票据状态 |
| 管理员 | 用户管理（CRUD/密码重置/冻结），票据审核（通过/退回） |

## 功能需求摘要

### 普通用户

| 编号 | 功能 | 说明 |
|------|------|------|
| FR-01 | 用户注册 | 自行注册，自动分配普通用户角色 |
| FR-02 | 添加票据 | 填写票据信息，"保存草稿"或"提交" |
| FR-03 | 修改草稿 | 仅可编辑状态为"草稿"的本人票据 |
| FR-04 | 删除草稿 | 仅可删除状态为"草稿"的本人票据 |
| FR-05 | 查看票据列表 | 按状态筛选（全部/草稿/待审核/已通过/已退回） |
| FR-06 | 查看票据详情 | 查看指定票据的完整信息 |
| FR-07 | 搜索票据 | 按关键词、类型、日期范围搜索 |

### 管理员

| 编号 | 功能 | 说明 |
|------|------|------|
| FR-08 | 票据审核 | 对已提交票据做出"通过"或"退回"决定 |
| FR-09 | 审核意见 | 审核时填写意见（必填，尤其退回时） |
| FR-10 | 用户管理 | 用户 CRUD、密码重置、账号冻结/解冻 |

## 票据状态流转

```
草稿 → 已提交 → 已通过
              ↘ 已退回 → (用户可重新提交)
```

详见 [[票据状态流程]]

## 非功能需求

- 前端使用 Vue 3 + Element Plus
- 后端基于 RuoYi-Vue 框架
- 前后端分离，API 通过 HTTP 通信
- 鉴权使用 Spring Security + JWT
- 数据库使用 MySQL 8.0
